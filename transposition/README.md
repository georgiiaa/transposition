# transposition

*the richer language beneath the poorer medium*

A hermeneutic reading tool for scripture. Enter a reference and the verse
returns in its original language — Greek, Hebrew, or Aramaic — with the
English translation set small underneath it, where it belongs. Hover over any
word to see the single English gloss a translator would have to settle for;
open the word to see everything it actually carries: lemma, morphology,
Strong's number, semantic range, etymology, where else it works differently
in scripture, and a note on what the standard English rendering loses.

The name is borrowed from C.S. Lewis's essay "Transposition," his account of
what happens when a richer language is rendered into a poorer medium. The
richer meaning does not vanish, but it does wait to be looked for.

## How it works

- **Frontend** — React + Vite, one page, no framework beyond that. White
  ground, [International Klein Blue](https://en.wikipedia.org/wiki/International_Klein_Blue)
  for everything interactive.
- **Backend** — a single Vercel serverless function (`api/claude.js`) that
  holds the Anthropic API key and builds all prompts server-side. The client
  sends structured requests, never raw prompts.
- **Philology** — verse retrieval and word studies are generated by Claude.
  Word-level lookups are verified at request time against STEPBible, Blue
  Letter Bible, and standard lexicons (BDAG/LSJ, BDB/HALOT) via web search;
  each entry reports which source confirmed the parsing, or names the
  discrepancy if one was found.

## Running locally

```bash
npm install
npm install -g vercel   # if you don't have it
cp .env.example .env    # then add your Anthropic API key
vercel dev              # runs frontend + api/ function together
```

`npm run dev` alone runs only the frontend; the `/api/claude` endpoint needs
`vercel dev` (or a deployment) to exist.

## Deploying

1. Push this repository to GitHub.
2. Import it at [vercel.com/new](https://vercel.com/new) — Vercel detects
   Vite and the `api/` directory automatically.
3. In the project settings, add an environment variable named
   `ANTHROPIC_API_KEY` with a key from
   [console.anthropic.com](https://console.anthropic.com).
4. Deploy.

## Roadmap

- Graft in [STEPBible's TAGNT/TAHOT datasets](https://github.com/STEPBible/STEPBible-Data)
  (CC BY 4.0) server-side, so morphological parsing comes from tagged data and
  the model writes only the interpretive layer — the semantic range, the
  account of loss.
- Verse-level caching so repeated references don't re-generate.
- Chapter view.

## A note on trust

A language model doing philology is fluent but occasionally wrong about
fine-grained morphology — which is, itself, a demonstration of this project's
premise. Verification against lexicons narrows the gap; it does not close it.
Check the sources. That is rather the point of the tool.
